Docker and Databases

These days containerisation is becoming increasingly common but is the same possible for databases? Does Docker support running a database?

I am sure the same question might have crossed your minds as well, not to worry as I have summarised it already for you all.

The data inside a container is ephermal which makes running stateful workloads difficult. However Docker provides tools (volume & bind mounts) that help in dealing with the state, allowing mounting a location on the host machine to a location in the container where one can store a Database.

How to do it? Simple, just the following :
  1. docker volume create my-volume
  2. docker run --mount source=my-volume,target=/app
  3. docker run ~/nginxlogs:/var/log/nginx

In AWS, it can be done with EFS (Details). Though Docker provides easy scaling, easier CI/CD, codification of the infrastructure, it is not usually recommended for database workloads as the complications exceed the benefits (in production environments). At development stages the same can prove to be useful to test and try different DB variations.

In summary, to save yourself from unnecessary complications, running the DB on a standard VPS or using AWS RDS would be a better choice (LINK).
